# Examples

## Trigger examples

- "Map this repo for me before I start changing the auth flow."
- "Give me a fast onboarding guide to this codebase, including where tests and entrypoints live."

## Expected output shape

- one-paragraph system summary
- key entrypoints and commands
- important directories or services
- risky modules or boundaries
- unanswered questions that need confirmation

## Discovery prompts

- Which manifest files define the runtime and tooling?
- Where does a request enter the system?
- Which tests cover the main business path?
